# Evaluación intermedia módulo 3 [Adalab](www.adalab.es)
## Ejercicio de evaluación intermedia del módulo 3 de Adalab: Desarrollo de una página web con React, en la que salgan renderizados un listado de Pokemon. 

![FireShot Capture 012 - Pokedex_ ¡Hazte con todos! - martreyz github io](https://user-images.githubusercontent.com/69849664/101240427-af3c9980-36ef-11eb-9da7-16a065f5b7a7.png)

### Descrición del proyecto:

La información de los Pokemon se toma de un array de objetos que debe guardarse en el estado del componente raíz: App.js
Proyecto creado utilizando HTML, CSS, JS y React.

### Arranque del proyecto:

- npm install: Para instalar las dependencias
<img width="624" alt="install" src="https://user-images.githubusercontent.com/69849664/101142786-61a02e00-3616-11eb-877a-cff3666ceee5.PNG">

- npm start: Para arrancar el proyecto y probarlo en desarrollo a través de la URL '//localhost:3000/#/'
<img width="625" alt="start" src="https://user-images.githubusercontent.com/69849664/101142795-649b1e80-3616-11eb-9c46-ea5a31cef5a2.PNG">

- npm run docs: Para publicar el proyecto a producción
<img width="628" alt="docs" src="https://user-images.githubusercontent.com/69849664/101142807-66fd7880-3616-11eb-9b84-2c1ecd7a9672.PNG">

### Composición del código:

Además del componente principal, se han creado dos componentes: Pokemon.js y PokeList.js, respectivamente para la tarjeta de cada pokemon y el listado de las mismas.

Como bonus del ejercicio, se nos pide que maquetemos las tarjetas. Esta maquetación se ha realizado en SASS y se divide en cuatro ficheros relacionados con cada uno de los componentes anteriormente indicados.

# Intermediate evaluation of module 3 of [Adalab](www.adalab.es)
## Intermediate evaluation exercise of the module 2 in Adalab consisting in developing a web site with React, where to render a list of Pokemon. 

![FireShot Capture 012 - Pokedex_ ¡Hazte con todos! - martreyz github io](https://user-images.githubusercontent.com/69849664/101240427-af3c9980-36ef-11eb-9da7-16a065f5b7a7.png)

### Description of the project:

The Pokemon's information is taken from an array of objects that must be saved in the state of the root component: App.js.
Project created using HTML, CSS, JS and React.

### How to start the proyect:

- npm install: To install dependencies
<img width="624" alt="install" src="https://user-images.githubusercontent.com/69849664/101142786-61a02e00-3616-11eb-877a-cff3666ceee5.PNG">

- npm start: To start the project and try it in development server through URL '//localhost:3000/#/'
<img width="625" alt="start" src="https://user-images.githubusercontent.com/69849664/101142795-649b1e80-3616-11eb-9c46-ea5a31cef5a2.PNG">

- npm run docs: To publish project to production
<img width="628" alt="docs" src="https://user-images.githubusercontent.com/69849664/101142807-66fd7880-3616-11eb-9b84-2c1ecd7a9672.PNG">

### Code composition:

Apart from root component, two additional components have ben created: Pokemon.js and PokeList.js, respectively for the card of each pokemon and the list of the same.

As an exercise bonus, it is required to mock up the cards. This mock-up has been realized in SASS and divided in four files related with each one of the components previously detailed.
